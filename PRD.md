# Kuclaw PRD

## 文档目标

本文件用于沉淀 Kuclaw 当前已经确认的产品需求、已完成能力与明确待办项。
后续当某项功能“复测通过”时，应同步把稳定结论更新到本文件，而不是只记录在对话中。

## 产品范围

Kuclaw 当前聚焦于“冻结式多屏截图选区器”主链路，目标是提供一套接近微信 / Snipaste 的桌面截图体验：

- 全局触发截图
- 冻结当前多屏桌面画面
- 智能窗口命中与高亮
- 未命中窗口时整屏 fallback
- 自由框选
- 确认后选区二次调整
- 复制 / 保存 / 取消
- 多屏下工具栏、保存面板与交互归属正确
- 尺寸 / 坐标提示与真实像素放大镜

## 当前已稳定能力

### 冻结式截图主链路

- 支持冻结桌面后进入截图态，避免 live 桌面变化影响选区判断。
- 支持基于缓存窗口候选做 hover 命中，不在鼠标移动阶段重复做原生窗口枚举。
- 支持命中窗口时高亮窗口区域并单击确认。
- 支持未命中任何窗口时，自动回退为“当前物理屏整屏高亮”，并支持单击整屏截图。

### 多屏与交互

- 支持多显示器截图，每块物理屏使用独立 overlay。
- 主屏和副屏都可正常 hover 高亮并截图。
- 保存面板会跟随实际触发保存动作的 overlay，出现在正确屏幕 / Space。
- 当命中窗口底部接近系统保留区域（如 macOS Dock）时，工具栏会自动避让并优先翻到选区上方，避免按钮被 Dock 或系统保留区遮挡。

### 选区能力

- 支持自由框选任意区域。
- 支持确认后的二次调整：
  - 拖动整块选区
  - 拖动四边拉伸
  - 拖动四角拉伸
- 支持 8 个控制点与对应鼠标光标反馈。

### 结果输出

- 支持工具栏动作：
  - 复制
  - 保存
  - 取消
- 复制 / 取消后主界面不会再异常弹回并抢焦点。
- Retina / 高 DPI 屏下导出的截图会保留清晰文字与细节，不会因为冻结底图降采样而明显发糊。
- 命中窗口时，截图态信息面板会显示当前窗口所属应用名。
- 保存截图时，默认文件名会尽量带上命中窗口的应用名，便于区分不同来源截图；如果用户后续手动调整选区，则按自由框选语义处理。

### 辅助反馈

- 支持尺寸与坐标提示面板，实时显示当前选区的 `宽 x 高` 与左上角坐标。
- 按下 `F1` 进入截图态后，在未开始框选前，坐标提示中的 `X / Y` 会跟随鼠标指针实时变化。
- 支持颜色值提示与取色：信息面板会显示当前指针颜色值，按 `C` 可复制当前颜色的 `#RRGGBB`，且不打断截图会话。
- 支持颜色值格式切换：截图态按 `Shift` 可在 HEX 与 RGB 之间切换显示与复制格式；默认格式可在设置中配置，默认显示 RGB。
- 支持真实像素放大镜，基于冻结底图显示局部放大、网格与中心十字线。
- 放大镜支持设置开关，默认开启；关闭后仅隐藏放大镜，不影响 hover、高亮、自由框选与结果导出。
- 主界面支持“最近复制颜色”列表，记录当前应用会话内最近的颜色复制结果，包含复制时间、坐标值、颜色值和颜色块示例。
- 最近复制颜色列表中的颜色值支持 hover 高亮与点击复制：鼠标悬停时显示小手，点击后会再次复制该颜色值，并显示短暂的亮绿反馈动画。

### 主窗口与原生标题栏

- 在 macOS 主窗口普通窗口态下，左上角保留原生 `traffic lights`，`Toggle Sidebar + Back + Forward` 显示在其右侧并与其保持同一排。
- 普通窗口态下，`Toggle Sidebar` 支持正常 hover 与点击，返回 / 前进作为同一组导航控件可见。
- 普通窗口态下，顶部空白区支持拖动窗口，不再额外出现空白工具栏分隔线。
- 在 macOS 主窗口 fullscreen 态下，`Toggle Sidebar + Back + Forward` 默认隐藏；当鼠标移到屏幕顶部时，会跟随系统 reveal band 从顶部滑出，并保持可点击。
- fullscreen 态下仍保留系统原生 `traffic lights` 交互，同时不再出现只剩分隔线或空工具栏的视觉残留。

### 菜单栏与后台入口

- 在 macOS 顶部系统菜单栏右侧提供 Kuclaw 的原生 Menu Bar Extras 入口。
- 菜单栏图标统一使用 `icon.icns` 作为唯一资源源，不再维护单独的菜单栏专用图标文件。
- 菜单栏图标会按当前菜单栏所在屏幕的显示倍率重新栅格化，减少在不同显示器 / Retina 倍率切换时的明显失真。
- 菜单栏状态项继续使用原生 macOS 状态项承载，而不是回退到 Qt 的通用托盘实现。

## 当前明确待办

### 暂不做

- 确认后选区可跨屏调整。
  - 当前确认后的选区编辑仍以“单个 overlay 内可见且可理解”为边界。
  - 暂不支持把一个已确认选区继续拖动或拉伸到另一块物理屏上。
  - 原因：当前每块物理屏由独立 `FreezeOverlayWidget` 承载，若直接开放跨屏调整，会出现“导出区域已跨屏，但只有发起编辑的 overlay 在绘制可视反馈”的不一致问题。
  - 后续若要实现，需要先统一多屏 overlay 之间的已确认选区共享状态与同步重绘机制。

### 后续优先事项

- 工具栏扩展能力，例如贴图等后续动作
- PRD 与开发记录的自动同步流程进一步标准化
- 在不改变当前普通窗口 / fullscreen 交互的前提下，继续收敛 macOS 原生标题栏承载方式，降低不同窗口状态下的位置漂移和命中区维护成本。
- 在保持 `icon.icns` 作为唯一源的前提下，继续优化菜单栏图标的小尺寸清晰度，尤其是 `16/32/64` 级别 representation 的锐度表现。

## 更新规则

- 当某项功能完成开发并复测通过后：
  - 将“用户可感知的稳定行为”整理到“当前已稳定能力”
  - 若产生明确不做或延期项，则更新到“当前明确待办”
  - 避免只写实现细节，不写产品行为
