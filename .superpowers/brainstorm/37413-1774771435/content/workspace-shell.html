Kuclaw macOS shell direction

Below is the recommended shell direction for the main window before we move to Figma refinement. It keeps the interface native to macOS, leaves unspecified content blank, and uses the toolbar plus sidebar behavior you confirmed.

A. Auto-hide rail + toggle expand

Default state is clean and content-first. Hovering the left edge reveals a narrow icon rail, while the toolbar toggle temporarily expands the full sidebar until toggled closed again.

Recommended

B. Persistent icon rail

Keeps navigation always visible, but reduces the calm, native macOS feel and takes horizontal space even when the user is focused on content.

C. Always-open full sidebar

Simplest behavior, but visually heavier and less aligned with your requirement that the shell should feel minimal until users intentionally reveal navigation.
