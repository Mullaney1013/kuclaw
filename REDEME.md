## 2026-03-26

### 已实现功能

- 调整“截图时保留当前窗口”的交互语义：现在会先用主界面可见状态完成冻结截图，再在 overlay 显示前把主界面从 live window 栈里临时隐藏；因此主界面仍会出现在截图底图中，但在截图交互阶段不再参与焦点竞争，避免点击工具栏按钮后系统把焦点还给主界面。
- 继续收口“截图动作后主界面回弹抢焦点”的业务链路：新增 `captureCompleting` 前置信号，把 reopen 抑制覆盖到复制/保存完成前；同时移除 `captureFinished -> restoreMainWindow()` 的 QML 显式恢复链路，并将 reopen 抑制时长延长到 1200ms，用于覆盖 overlay 关闭后连续触发的多次应用激活事件。
- 继续修正截图取消后的主界面回弹链路：确认根因是工具栏“取消”会触发多次 `ApplicationActivate / ApplicationStateChange`，而旧抑制逻辑只吞掉第一次事件；现已改为在抑制时间窗内持续忽略这类 reopen 事件，不再在第一下激活后立刻解除抑制。
- 根据 review 的 P1 问题收紧截图取消恢复逻辑：为截图临时隐藏的窗口现在会记录“是否主界面 + 原始可见状态”，取消截图时恢复非主窗口（如 pin 等辅助顶层窗）且按原状态恢复，成功完成或异常退出时仍恢复全部窗口；避免取消截图后把其他 Kuclaw 顶层窗口永久丢失。
- 继续收紧截图取消时的主界面回弹抑制：新增 `captureCanceling` 前置通知，在关闭冻结遮罩前就提前抑制一次 `reopenRequested`，用于覆盖工具栏“取消”按钮导致的应用激活事件抢跑问题。
- 修正冻结式截图工具栏“取消”导致主界面抢焦点的问题：截图取消现在会短暂抑制一次 `reopenRequested`，避免工具栏按钮点击引发的应用激活事件把 Kuclaw 主界面重新弹回前台；真正的 Dock 点击恢复链路仍保留。
- 调整截图取消交互：当启用“截图时不保留当前窗口”时，点击冻结式截图工具栏的“取消”或按 `Esc` 退出截图，不再自动把 Kuclaw 主界面重新 show / raise / activate 回前台；主界面恢复仅保留在截图成功完成后的路径。
- 恢复原生冻结式截图中的“任意区域自由框选”能力：冻结层现在支持左键按下后拖拽任意矩形区域，松开即可直接截图；若只是轻点已高亮窗口，则仍按窗口命中结果快速截图。
- 去掉全局 `Esc` 热键注册，截图取消现在只通过应用内事件通道和冻结层自身的 `Esc / 右键` 处理，不再影响其他应用的 `Escape` 键。
- 将原生冻结式截图遮罩从“单个跨虚拟桌面的 QWidget”改为“每块物理屏一个 FreezeOverlayWidget”，并把局部鼠标坐标统一映回虚拟桌面坐标后再做窗口命中，专门修正 macOS 双屏下只有其中一块屏能持续收到 hover 高亮的问题。
- 进一步修正 macOS 多屏冻结截图的显示器对齐：`CGDisplay/NSScreen` 现在会优先绑定到 Qt 的 `QScreen::geometry()` 与 `devicePixelRatio()`，让覆盖层、鼠标坐标和窗口候选统一落在同一套 Qt 逻辑坐标系里。
- 修正 macOS 多屏命中在“MacBook Retina 内屏 + 外接副屏”场景下的坐标优先级：窗口命中现在优先按 Cocoa/Qt 点坐标屏幕框做映射，仅在未命中任何点坐标屏幕时才回退到像素坐标，避免内屏窗口被额外缩放导致只能命中副屏窗口。
- 进一步加强 macOS 冻结式截图的窗口坐标映射：新增按显示器拓扑的 `topLeftFrame -> logicalRect` 转换，命中窗口矩形不再直接裸用 `CGWindowBounds`，为多屏和缩放场景做了额外校准。
- 修正 macOS 冻结式截图的窗口命中坐标：移除了对 `CGWindowList` 返回窗口矩形的重复 Y 翻转，窗口 hover 高亮与最终截图区域现在按左上角坐标系直接映射。
- 新增 `scripts/run-kuclaw-mac.sh` 启动脚本：支持在本机 Aqua GUI 会话中一键重启并启动 Kuclaw，默认使用 `open` 拉起 `.app`，并可通过 `--build` 先构建、通过 `--direct` 直接运行二进制查看日志。
- 将新的“冻结式多屏截图选区器”正式接入现有工程：新增 `CaptureCoordinator + FreezeOverlayWidget + INativeScreenHelper` 抽象层，并把截图启动链路改为先冻结桌面、再显示原生顶层遮罩、再基于冻结时缓存的窗口矩形做 hover 命中。
- 新增 Windows 原生冻结截图后端骨架：接入 `RegisterHotKey / WM_HOTKEY`、`QScreen::grabWindow(0)` 多屏拼图、`EnumWindows` 顶层窗口缓存与逻辑坐标换算。
- 新增 macOS 原生冻结截图后端骨架：接入 ScreenCaptureKit 截图链路（`SCScreenshotManager` + `SCStream` fallback）、`CGWindowListCopyWindowInfo` 候选窗口枚举，以及屏幕录制权限检查。
- 主界面截图入口已切换到新的 `appCoordinator.beginCapture()`，不再依赖旧的 QML 遮罩实例化流程；界面状态文案也改为展示原生冻结截图后端状态。

### 已优化功能

- 将截图遮罩从主窗口内部层改为独立顶层窗口，更接近真实截图工具行为。
- 修正主窗口隐藏导致截图遮罩可能被连带隐藏的问题：将遮罩窗口从 `Qt.Tool` 调整为独立 `Qt.Window`，并为截图启动增加失败兜底计时器，防止进入截图态后主窗口长期消失。
- 修订截图启动流程：`F1` 先触发会话启动，再在遮罩窗口可见时才隐藏主窗口，避免“主窗消失但无遮罩”的错误状态；并为遮罩窗口补充几何兜底（无截图几何时默认到主屏尺寸）。
- 补充桌面几何与截图快照在 ViewModel 中的暴露方式，解决多屏坐标和截图展示的基础承接问题。
- 修正选区缩放逻辑，避免缩小过程中出现非法尺寸或反向翻转。
- 调整截图提示文案，使当前可用快捷操作与实际行为一致。
- 将截图流程中的主窗口隐藏与恢复接入，减少把主窗口截进去的概率。
- 对项目骨架做了一轮静态一致性修正，补齐部分 Qt 头文件依赖和属性声明位置问题。
- 优化截图遮罩交互：已有选区时，按住鼠标左键不会强制重建选区，而是可以配合键盘继续微调边缘。
- 补充放大镜图像链路，由 `C++ -> ViewModel -> QML` 传递局部像素快照。
- 增加 hover 性能可观测性开关：`KUCLAW_CAPTURE_HOVER_LATENCY`、`KUCLAW_CAPTURE_HOVER_LATENCY_WINDOW_MS`、`KUCLAW_CAPTURE_HOVER_LATENCY_SLA_MS`，并在日志中直接标注 `meets` 和优化建议。
- 调整冻结式截图 hover 行为：当鼠标未命中任何底层窗口时，自动回退为高亮当前物理屏幕整屏，并支持单击直接确认整屏截图。
- 修正冻结式截图的保存对话框父窗口选择逻辑，多屏场景下 `保存` 面板会跟随实际触发保存动作的那块 overlay。
- 为冻结式截图补齐选区二次调整：确认选区后支持拖动整块区域、拖动四边/四角进行拉伸，并显示 8 个控制点与对应鼠标光标。
- 增加冻结式截图的尺寸与坐标提示面板，在 hover、自由框选和二次调整阶段实时显示当前选区的 `宽 x 高` 与左上角坐标。
- 增加冻结式截图的真实放大镜功能，基于冻结底图对当前鼠标附近像素做局部放大，并叠加网格与中心十字线用于精细对齐。
- 调整 macOS 高 DPI 冻结截图的物理像素链路：不再使用 `CGDisplayBounds` 推导每块屏的导出像素尺寸，而是直接以 ScreenCaptureKit 实际返回的单屏 `frame` 像素大小建立 `displaySegments`，并据此进行冻结底图拼接、放大镜取样和最终截图导出，避免 Retina 屏截图再次被压回逻辑尺寸。
- 将冻结式截图的放大镜改为可配置项，默认开启；设置面板新增“截图放大镜”开关，关闭后仅隐藏放大镜，不影响 hover、高亮、自由框选与结果导出链路。
- 调整坐标提示面板的触发逻辑：按下 `F1` 进入截图态后，在未开始框选前，`X / Y` 会随鼠标移动实时刷新，不再依赖 hover 区域变化才更新。
- 为冻结式截图补充颜色值提示与取色：信息面板新增当前指针颜色值显示，按 `C` 可将当前颜色的 `#RRGGBB` 写入系统剪贴板，且不会中断截图会话。
- 为颜色值提示补充格式切换能力：截图态按 `Shift` 可在 HEX 与 RGB 之间切换显示与复制格式；同时新增“颜色值默认格式”设置项，默认显示 RGB。
- 为按 `C` 复制颜色值补充交互反馈：信息面板中的 `C` 目标提示在触发复制后会立即加粗并高亮为亮绿色，保持约 200ms 后再平滑淡回默认状态，增强取色成功的感知。
- 增加“最近复制颜色”列表：当截图态按 `C` 复制颜色值后，主界面右侧会追加一条最近颜色记录，包含复制时间（年月日时分秒）、坐标值、颜色值和颜色块示例；当前实现按应用会话内最近 20 条记录保留。
- 扩展 Windows 全局热键解析范围：`RegisterHotKey` 现在支持字母、数字、`F1-F24` 与常见导航键（如 `Tab`、方向键、`Home/End/PageUp/PageDown` 等），并会对 Windows 不支持的多段快捷键给出明确错误提示，便于后续在 Windows 真机上验证可配置热键。
- 新增 `scripts/deploy-windows.bat` 部署脚本：支持在 Windows 上对 `Kuclaw.exe` 运行 `windeployqt`，并可选先执行 CMake configure/build，最终产出一个可直接拿去做冒烟测试的运行目录。
- 调整 Windows 部署脚本的默认构建目录为 `build-windows`，避免将 macOS 上复制过来的 `build` 缓存误当作 Windows 构建目录继续复用。
- 为冻结式截图补充命中窗口的应用识别链路：`WindowCandidate` 现在会携带应用名（macOS 读取 `kCGWindowOwnerName`，Windows 根据 `ownerPid` 解析进程名并附带窗口标题），截图态信息面板新增“应用”一行，同时保存截图时的默认文件名会尽量带上应用名，便于区分不同来源窗口。
- 调整主界面结构为更接近 Codex 的左侧导航布局：主界面改成“工作台 / 贴图 / 颜色 / 设置”四个一级入口，首页只保留动作入口与状态卡片；同时将截图相关参数收进设置页内部菜单栏，并按“截图 / 取色 / 快捷键 / 存储”重新分组。
- 继续打磨主界面视觉层级：强化左侧导航的选中态与信息卡片节奏，调整工作台首页的 Hero 卡片、状态胶囊和摘要卡片，让整体更接近 Codex 的控制台式交互气质；同步提升设置页菜单栏的层级感与分区标签样式。
- 按 Figma 设计稿重构主界面交互：工作台首页改成更接近 Codex 的中心化 Hero + 建议卡片 + 输入面板结构；左下角 `Settings` 改为先弹出账号/应用菜单，再进入独立设置页；新增正式的 `Settings > Capture` 页面结构，并将截图相关配置重新挂回该页面。
- 对齐最新 Figma 稿继续收口主界面与设置页：工作台首页移除上一轮临时加入的说明卡片，回到更纯的中心化 `Let’s capture` 结构；左下角 `Settings` 弹层更新为最新的账号信息与菜单文案；`Settings` 详情页同步调整为 `Capture Settings` 导航命名，并补齐顶部 `Settings` 标题区与最新分组层级。
- 按“三张页面一步一步完成”的方式先单独对齐 `Kuclaw Workspace Concept`：工作台首页的中心 Hero、三张建议卡片、输入面板和左侧工作区列表继续向最新 Figma 稿收口，避免与 `Settings Entry / Settings Detail` 的后续改动混在一起。
